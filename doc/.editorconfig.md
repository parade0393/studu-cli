# `.editorconfig` 说明（本项目）

## 1) `.editorconfig` 文件介绍

`.editorconfig` 用来在不同编辑器/IDE 之间统一基础格式规范（缩进、换行、编码、是否去除行尾空白等）。支持 EditorConfig 的工具会：

- 从当前文件所在目录向上查找 `.editorconfig` 并按顺序合并规则
- 按 section 的“路径匹配”选择适用规则，越具体的 section 会覆盖更通用的设置
- 将这些设置应用到保存/格式化行为（具体是否强制取决于编辑器/插件/格式化器）

## 2) 本项目的 `.editorconfig` 配置解析

本项目根目录 `.editorconfig` 内容如下（按实际生效顺序解析）。

### `root = true`

表示“从此处作为配置根”，编辑器/插件不再继续向上查找父目录的 `.editorconfig`，避免受到外部目录配置影响。

### `[*]`（对所有文件生效的默认规则）

该 section 匹配所有文件，并设置默认格式：

- `charset = utf-8`：文件编码使用 UTF-8
- `indent_style = space`：使用空格缩进（不使用 tab）
- `indent_size = 2`：缩进宽度为 2 个空格
- `end_of_line = lf`：行尾使用 LF（`\n`）
- `insert_final_newline = true`：文件末尾保证有一个换行
- `trim_trailing_whitespace = true`：保存时移除行尾多余空白

### `[*.md]`（Markdown 特例）

只对 `*.md` 生效：

- `trim_trailing_whitespace = false`：不移除行尾空白

该设置常用于保留 Markdown 的“行尾两个空格表示强制换行”等语义。

### `[*.{json,yml,yaml}]`（JSON/YAML 缩进）

对 `*.json` / `*.yml` / `*.yaml` 生效：

- `indent_size = 2`：缩进宽度为 2 个空格

由于 `[*]` 已经设置了 `indent_size = 2`，这里属于显式重申；当你未来调整默认缩进时，这个 section 也能确保 JSON/YAML 仍保持 2 空格。

## 自查命令（可选）

如果你使用的是支持 EditorConfig 的环境，也可以用编辑器的状态栏/插件查看当前文件命中的规则；或在项目中搜索确认是否有其他工具（如 Prettier/ESLint）对同类格式有更强的覆盖。
