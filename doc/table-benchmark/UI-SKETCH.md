## /picking（波次拣货工作台：左右两表 + 强编辑）

+--------------------------------------------------------------------------------------------------+
| Filters (影响左表): 仓库/库区/关键字/状态/效期 |
| [查询] [重置筛选] |
+--------------------------------------------------------------------------------------------------+
| LeftTable: 可拣库存 (45%) | Actions (10%) | RightTable: 拣货明细 (45%)|
| - selection 多选 | [生成拣货行 >>] | - row editable |
| - 可选：tree 展开 | [<< 移除选中明细] | - validators + async |
| - 列较多但读为主 | [批量设置 picker] | - 批量/向下填充/撤销 |
| | [批量设置 strategy] | - rowStatus + errors |
| | [触发全量校验] | |
| | [提交 ready 行] | |
+--------------------------------------------------------------------------------------------------+
| BottomSummary |
| 错误行：xx | 待提交(ready)：xx | pickQty 合计：xx | 最近一次提交耗时：xxms | [导出当前明细JSON] (可选) |
+--------------------------------------------------------------------------------------------------+

右表示意（关键列）：

- 来源信息：sku / skuName / batch / bin / available
- 拣货信息：pickQty(数字编辑) / picker(远程搜索下拉) / strategy(下拉) / remark(textarea)
- 状态信息：rowStatus badge / errors 摘要 / 操作(编辑/保存/取消/撤销)

交互提示：

- 单元格错误：红框 + tooltip
- 行状态：dirty/validating/error/ready/submitted
- 异步校验：行内显示“校验中...”并禁止提交
